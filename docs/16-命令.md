# 自定义命令

为重复任务创建自定义命令。

## 创建命令

`.opencode/commands/test.md`：

```markdown
---
description: Run tests with coverage
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

Run the full test suite with coverage report.
Focus on the failing tests and suggest fixes.
```

使用：

```
/test
```

## 命令位置

- 全局：`~/.config/opencode/commands/`
- 项目级：`.opencode/commands/`

## JSON 配置

```json
{
  "command": {
    "test": {
      "template": "Run the full test suite with coverage...",
      "description": "Run tests with coverage",
      "agent": "build",
      "model": "anthropic/claude-3-5-sonnet-20241022"
    }
  }
}
```

## 提示词配置

### 参数

```markdown
---
description: Create a component
---

Create a new React component named $ARGUMENTS with TypeScript.
```

使用：

```
/component Button
```

位置参数：`$1`、`$2`、`$3`...

### Shell 输出

```markdown
---
description: Analyze coverage
---

Here are the test results:
!`npm test`
Based on these results, suggest improvements.
```

### 文件引用

```markdown
---
description: Review component
---

Review the component in @src/components/Button.tsx
```

## 选项

| 选项 | 说明 |
|------|------|
| `template` | 发送给 LLM 的提示词（必需） |
| `description` | 命令描述 |
| `agent` | 使用的代理 |
| `subtask` | 强制子代理调用 |
| `model` | 覆盖默认模型 |

## 覆盖内置命令

自定义命令可覆盖内置命令（如 `/init`）。

## 示例

```markdown
---
description: Create new component
agent: build
---

Create a new React component named $ARGUMENTS with:
- TypeScript support
- Proper typing
- Basic structure
```
