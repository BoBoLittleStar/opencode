# 权限控制

使用 `permission` 配置控制哪些操作需要审批。

## 操作

| 操作 | 说明 |
|------|------|
| `allow` | 无需审批直接运行 |
| `ask` | 提示审批 |
| `deny` | 阻止操作 |

## 全局配置

```json
{
  "permission": {
    "*": "ask",
    "bash": "allow",
    "edit": "deny"
  }
}
```

或一次性设置：

```json
{
  "permission": "allow"
}
```

## 细粒度规则

```json
{
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "rm *": "deny"
    },
    "edit": {
      "*": "deny",
      "packages/web/src/**/*.mdx": "allow"
    }
  }
}
```

### 通配符

- `*` - 零个或多个任意字符
- `?` - 精确一个字符

## 外部目录

允许工具访问项目工作目录之外的路径：

```json
{
  "permission": {
    "external_directory": {
      "~/projects/personal/**": "allow"
    }
  }
}
```

## 主目录展开

- `~` → `/Users/username`
- `$HOME` → `/Users/username`

## 可用权限

| 权限 | 说明 |
|------|------|
| `read` | 读取文件 |
| `edit` | 文件修改（edit/write/patch/multiedit）|
| `glob` | 文件通配 |
| `grep` | 内容搜索 |
| `list` | 列出目录 |
| `bash` | shell 命令 |
| `task` | 子代理 |
| `skill` | 技能加载 |
| `lsp` | LSP 查询 |
| `webfetch` | 获取 URL |
| `websearch` | 网页搜索 |
| `external_directory` | 外部路径 |
| `doom_loop` | 重复工具调用检测 |

## 默认值

- 大多数权限默认为 `allow`
- `doom_loop` 和 `external_directory` 默认为 `ask`
- `.env` 文件默认被拒绝

## 审批选项

- `once` - 仅本次
- `always` - 记住模式
- `reject` - 拒绝

## 代理权限覆盖

```json
{
  "permission": {
    "bash": "*": "ask"
  },
  "agent": {
    "build": {
      "permission": {
        "bash": {
          "git *": "allow"
        }
      }
    }
  }
}
```

## Markdown 代理权限

```markdown
---
permission:
  edit: deny
  bash: ask
---

Only analyze code...
```
