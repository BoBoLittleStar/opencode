# 代理技能

通过 SKILL.md 定义可复用的行为。

## 文件位置

为每个技能创建一个文件夹，放入 `SKILL.md`：

- `.opencode/skills/<name>/SKILL.md`
- `~/.config/opencode/skills/<name>/SKILL.md`
- `.claude/skills/<name>/SKILL.md`（兼容）
- `.agents/skills/<name>/SKILL.md`（兼容）

## Frontmatter

每个 SKILL.md 必须以 YAML frontmatter 开头：

```markdown
---
name: git-release
description: Create consistent releases and changelogs
license: MIT
compatibility: opencode
metadata:
  audience: maintainers
  workflow: github
---

## What I do
- Draft release notes from merged PRs
- Propose a version bump
- Provide a copy-pasteable `gh release create` command
```

### 必需字段

- `name`：技能名称
- `description`：技能描述（1-1024 字符）

### 名称规则

```
^[a-z0-9]+(-[a-z0-9]+)*$
```

- 1-64 字符
- 小写字母和数字
- 单连字符分隔
- 不以 `-` 开头/结尾

## 发现机制

从当前目录向上遍历到 git 工作树根目录，加载匹配的 `skills/*/SKILL.md`。

全局从 `~/.config/opencode/skills/*/SKILL.md` 等位置加载。

## 工具描述

OpenCode 在 `skill` 工具描述中列出可用技能：

```
<available_skills>
  <skill>
    <name>git-release</name>
    <description>Create consistent releases</description>
  </skill>
</available_skills>
```

代理通过调用工具加载技能：

```
skill({ name: "git-release" })
```

## 权限配置

```json
{
  "permission": {
    "skill": {
      "*": "allow",
      "pr-review": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
```

| 权限 | 行为 |
|------|------|
| `allow` | 技能立即加载 |
| `deny` | 隐藏技能，拒绝访问 |
| `ask` | 加载前提示确认 |

## 按代理覆盖权限

### Markdown 代理

```markdown
---
permission:
  skill:
    "documents-*": "allow"
---
```

### 内置代理

```json
{
  "agent": {
    "plan": {
      "permission": {
        "skill": {
          "internal-*": "allow"
        }
      }
    }
  }
}
```

## 禁用技能工具

```markdown
---
tools:
  skill: false
---
```

或：

```json
{
  "agent": {
    "plan": {
      "tools": {
        "skill": false
      }
    }
  }
}
```

## 排查问题

1. 确认 `SKILL.md` 文件名全部大写
2. 检查 frontmatter 包含 `name` 和 `description`
3. 确保技能名称唯一
4. 检查权限设置
