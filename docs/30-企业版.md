# 企业版

在组织中安全使用 OpenCode。

## 概述

OpenCode 企业版面向希望确保代码和数据留在自有基础设施内的组织。通过集中式配置与 SSO 和内部 AI 网关集成。

> OpenCode 不会存储您的任何代码或上下文数据。

## 试用

OpenCode 是开源的，默认不存储代码或数据。可以直接开始试用。

### 数据处理

所有处理在本地完成，或通过直接 API 调用发送至 AI 提供商。

#### 分享对话

如果启用 `/share`，对话会发送到托管服务。建议试用期间禁用：

```json
{
  "share": "disabled"
}
```

### 代码所有权

您拥有 OpenCode 生成的所有代码，无许可限制。

## 定价

按席位定价。如有自有 LLM 网关，不收取 Token 费用。

## 部署选项

### 集中式配置

为整个组织设置统一配置，可与 SSO 提供商集成。

### SSO 集成

通过集中式配置与组织 SSO 提供商集成认证。

### 内部 AI 网关

配置 OpenCode 仅使用内部 AI 网关，可禁用其他提供商。

### 自托管

可帮助在自有基础设施上托管分享页面。

## 常见问题

### 我的数据安全吗？

是的。OpenCode 不存储代码或上下文数据。通过集中式配置和 SSO 集成，数据保留在组织基础设施内。

### 可以使用私有 NPM 注册表吗？

支持。通过 `.npmrc` 文件配置私有注册表：

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

或手动配置：

```bash
# ~/.npmrc
registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```
