# 故障排除

常见问题及解决方法。

## 日志

日志文件位置：
- **macOS/Linux**: `~/.local/share/opencode/log/`
- **Windows**: `%USERPROFILE%\.local\share\opencode\log`

日志以时间戳命名，保留最近 10 个文件。

```bash
opencode --log-level DEBUG
```

## 存储

- **macOS/Linux**: `~/.local/share/opencode/`
- **Windows**: `%USERPROFILE%\.local\share\opencode`

包含：
- `auth.json` - API 密钥、OAuth Token
- `log/` - 应用日志
- `project/` - 会话和消息数据

## 桌面应用

### 快速检查

- 完全退出并重新启动
- 点击"重新启动"并复制错误详情
- macOS：`OpenCode` 菜单 → `Reload Webview`

### 禁用插件

检查全局配置 `~/.config/opencode/opencode.jsonc`：

```json
{
  "plugin": []
}
```

临时移走插件目录：
- macOS/Linux: `~/.config/opencode/plugins/`
- Windows: `%USERPROFILE%\.config\opencode\plugins`

### 清除缓存

```bash
# macOS
rm -rf ~/Library/Caches/opencode

# Linux
rm -rf ~/.cache/opencode

# Windows
# 删除 %USERPROFILE%\.cache\opencode
```

### 修复服务器连接

1. 点击服务器名称 → "清除"默认服务器 URL
2. 移除配置中的 `server.port` / `server.hostname`
3. 取消设置 `OPENCODE_PORT` 环境变量

### Windows WebView2

安装或更新 Microsoft Edge WebView2 Runtime。

### Linux Wayland/X11

```bash
OC_ALLOW_WAYLAND=1 opencode
```

## 常见问题

### OpenCode 无法启动

1. 检查日志错误
2. `opencode --print-logs`
3. `opencode upgrade`

### 身份验证问题

1. `/connect` 重新认证
2. 检查 API 密钥
3. 检查网络连接

### 模型不可用

1. 检查提供商认证
2. 验证模型名称
3. 检查访问权限

### ProviderInitError

1. 清除存储：`rm -rf ~/.local/share/opencode`
2. 重新认证：`/connect`

### AI_APICallError

1. 清除提供商包：`rm -rf ~/.cache/opencode`
2. 重启 OpenCode

### Linux 复制/粘贴不可用

```bash
# X11
apt install -y xclip

# Wayland
apt install -y wl-clipboard
```
