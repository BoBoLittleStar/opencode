# LLM 提供商

OpenCode 使用 [AI SDK](https://ai-sdk.dev/) 和 [Models.dev](https://models.dev)，支持 **75+ LLM 提供商**，同时也支持运行本地模型。

## 添加提供商

1. 使用 `/connect` 命令添加提供商的 API 密钥
2. 在配置中设置提供商

凭据存储在 `~/.local/share/opencode/auth.json`

## 自定义 Base URL

```json
{
  "provider": {
    "anthropic": {
      "options": {
        "baseURL": "https://api.anthropic.com/v1"
      }
    }
  }
}
```

## 支持的提供商

### OpenCode Zen（推荐新手）

详见 [34-Zen.md](./34-Zen.md)

### OpenAI

1. `/connect` → 选择 OpenAI
2. 选择 **ChatGPT Plus/Pro** 或手动输入 API Key
3. `/models` 选择模型

### Anthropic

1. `/connect` → 选择 Anthropic
2. 选择 **Claude Pro/Max** 或手动输入 API Key
3. `/models` 选择模型

### Google Vertex AI

```json
{
  "provider": {
    "google": {
      "options": {
        "baseURL": "https://api.google.com/v1"
      }
    }
  }
}
```

需要设置：
- `GOOGLE_CLOUD_PROJECT`
- `VERTEX_LOCATION`（可选）
- 认证：`GOOGLE_APPLICATION_CREDENTIALS` 或 `gcloud auth`

### Amazon Bedrock

```json
{
  "provider": {
    "amazon-bedrock": {
      "options": {
        "region": "us-east-1",
        "profile": "my-aws-profile"
      }
    }
  }
}
```

认证方式（优先级）：
1. Bearer Token (`AWS_BEARER_TOKEN_BEDROCK`)
2. AWS 凭证链（配置文件、访问密钥、IAM 角色）

### Azure OpenAI

```bash
export AZURE_RESOURCE_NAME=xxx
```

### OpenRouter

通过 `/connect` 添加，默认预加载许多模型。

### DeepSeek

## 自定义提供商

### 添加 OpenAI 兼容提供商

1. `/connect` → 滚动到 **Other**
2. 输入提供商 ID（如 `myprovider`）
3. 输入 API 密钥
4. 在 `opencode.json` 中配置：

```json
{
  "provider": {
    "myprovider": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "My AI Provider",
      "options": {
        "baseURL": "https://api.myprovider.com/v1"
      },
      "models": {
        "my-model-name": {
          "name": "My Model Display Name"
        }
      }
    }
  }
}
```

### 高级配置示例

```json
{
  "provider": {
    "myprovider": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "My AI Provider",
      "options": {
        "baseURL": "https://api.myprovider.com/v1",
        "apiKey": "{env:ANTHROPIC_API_KEY}",
        "headers": {
          "Authorization": "Bearer custom-token"
        }
      },
      "models": {
        "my-model-name": {
          "name": "My Model",
          "limit": {
            "context": 200000,
            "output": 65536
          }
        }
      }
    }
  }
}
```

## 故障排除

1. 运行 `opencode auth list` 检查凭据
2. 确认配置中提供商 ID 与 `/connect` 中使用的 ID 一致
3. 确认使用了正确的 npm 包
4. 确认 `baseURL` 正确

详见 [31-故障排除.md](./31-故障排除.md)
