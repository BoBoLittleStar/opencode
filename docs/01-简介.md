# OpenCode 简介

OpenCode 是一个开源的 AI 编码代理，提供终端界面、桌面应用和 IDE 扩展等多种使用方式。

## 核心特性

- **多端支持**：TUI 终端界面、Web 浏览器、桌面应用、IDE 插件
- **75+ LLM 提供商**：支持 Anthropic、OpenAI、Google、DeepSeek 等主流模型
- **本地模型**：支持 Ollama、LM Studio 等本地模型
- **深度集成**：GitHub Actions、GitLab CI/CD、VS Code、Cursor 等
- **可扩展**：自定义工具、插件、MCP 服务器

## 前提条件

### 终端要求

要在终端中使用 OpenCode，你需要：

1. **现代终端模拟器**：
   - [WezTerm](https://wezterm.org) - 跨平台
   - [Alacritty](https://alacritty.org) - 跨平台
   - [Ghostty](https://ghostty.org) - Linux 和 macOS
   - [Kitty](https://sw.kovidgoyal.net/kitty/) - Linux 和 macOS

2. **LLM 提供商 API 密钥**：你需要有想要使用的 LLM 提供商的 API 密钥

## 配置提供商

配置 API 密钥以使用任意 LLM 提供商：

### 方式一：使用 OpenCode Zen（推荐新手）

1. 在 TUI 中运行 `/connect` 命令，选择 `opencode`
2. 登录 [opencode.ai/auth](https://opencode.ai/auth) 并添加账单信息
3. 复制 API 密钥并粘贴到终端
4. 运行 `/models` 查看推荐模型列表

### 方式二：其他提供商

详见 [04-提供商.md](./04-提供商.md)

## 初始化项目

配置好提供商后，导航到项目目录并初始化：

```bash
cd /path/to/project
opencode
/init
```

OpenCode 会分析项目并在根目录创建 `AGENTS.md` 文件。

> **提示**：应将项目的 `AGENTS.md` 文件提交到 Git，有助于 OpenCode 理解项目结构。

## 使用方式

### 提问

让 OpenCode 讲解代码库：

```
How is authentication handled in @packages/functions/src/api/index.ts
```

使用 `@` 键可以模糊搜索项目中的文件。

### 添加功能

1. **制定计划**：使用 `Tab` 键切换到计划模式
   ```
   <TAB>
   When a user deletes a note, we'd like to flag it as deleted in the database.
   ```
2. **迭代计划**：提供反馈或补充细节
3. **构建功能**：按 `Tab` 键切换回构建模式
   ```
   <TAB>
   Sounds good! Go ahead and make the changes.
   ```

### 直接修改

对于简单修改，直接让 OpenCode 实施：

```
We need to add authentication to the /settings route. Take a look at how this is
handled in the /notes route in @packages/functions/src/notes.ts and implement
the same logic in @packages/functions/src/settings.ts
```

### 撤销修改

使用 `/undo` 命令撤销修改：

```
/undo
```

OpenCode 会还原所做的修改。使用 `/redo` 重做。
