# HTTP API 服务器

`opencode serve` 命令启动无界面 HTTP 服务器，暴露 OpenAPI 端点。

## 启动服务器

```bash
opencode serve [--port <number>] [--hostname <string>] [--cors <origin>]
```

### 选项

| 选项 | 默认值 | 说明 |
|------|--------|------|
| `--port` | 4096 | 监听端口 |
| `--hostname` | 127.0.0.1 | 监听主机名 |
| `--mdns` | false | 启用 mDNS |
| `--mdns-domain` | opencode.local | mDNS 域名 |
| `--cors` | [] | CORS 来源 |

### 身份验证

```bash
OPENCODE_SERVER_PASSWORD=your-password opencode serve
```

## OpenAPI 规范

```
http://<hostname>:<port>/doc
```

## API 端点

### 全局

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /global/health | 健康检查 |
| GET | /global/event | 全局事件流 |

### 项目

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /project | 列出项目 |
| GET | /project/current | 当前项目 |

### 会话

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /session | 列出会话 |
| POST | /session | 创建会话 |
| GET | /session/:id | 获取会话 |
| DELETE | /session/:id | 删除会话 |
| PATCH | /session/:id | 更新会话 |
| POST | /session/:id/abort | 中止会话 |
| POST | /session/:id/share | 分享会话 |
| DELETE | /session/:id/share | 取消分享 |
| POST | /session/:id/summarize | 总结会话 |

### 消息

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /session/:id/message | 列出消息 |
| POST | /session/:id/message | 发送消息 |
| POST | /session/:id/prompt_async | 异步发送消息 |
| POST | /session/:id/command | 执行命令 |
| POST | /session/:id/shell | 运行 shell |

### 文件

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /find?pattern= | 搜索文本 |
| GET | /find/file?query= | 查找文件 |
| GET | /file?path= | 读取文件 |

### TUI

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /tui/append-prompt | 追加提示词 |
| POST | /tui/submit-prompt | 提交提示词 |
| POST | /tui/open-sessions | 打开会话选择器 |

### 认证

| 方法 | 路径 | 说明 |
|------|------|------|
| PUT | /auth/:id | 设置凭据 |

### 事件

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /event | SSE 事件流 |

## 工作原理

运行 `opencode` 时：
1. 启动 TUI
2. 启动服务器
3. TUI 作为客户端与服务器通信

这允许：
- 多个客户端连接
- 编程方式交互

## 连接到现有服务器

```bash
opencode attach http://localhost:4096
```
