# OpenCode Auto-Answer 插件需求文档

本文档描述 Auto-Answer 插件的业务流程需求，设计/开发过程中有任何问题，请先阅读本文档

## 插件需求如下：

### Part 1 服务 & 实体

1. 做一个后端服务
2. 服务只支持“问题”和“回答”两种实体的存取
3. 保存问题 & 保存回答：
    1. 服务一次可以保存一组“问题”，必须全体指定一个特定的 id 作为来源
    2. 一个问题可以有多个选择项，每个选择项有自己的标签和描述
    3. “问题”不可修改或删除，可以被“回答”，一个问题最多只能回答一次,不可修改，不可回答第二次
    4. 回答时按来源 id 回答，必须一次性回答当前来源所有未回答的问题，如果不满足则不予处理
    5. 自定义回答长度不能超过 100 字
4. 查询问题：
    1. 服务支持按来源 id 分组查询每个来源下所有未回答的“问题”以及对应的选择项，过滤掉已回答的问题
    2. 服务支持按单个来源 id 筛选查询问题
    3. 不支持仅查询回答，回答跟随问题返回，若没有则返回空

### Part 2 OpenCode 交互逻辑

1. 当 OpenCode 启动时从插件启动服务，如果服务已经启动则无需处理
2. 当 OpenCode 退出时**保持服务运行，不要关闭**
3. 当 OpenCode 使用 question 工具提问之前拦截所有问题，并发送给服务，服务保存问题
4. 保存问题时使用 OpenCode 的会话 id 作为问题来源，以允许 OpenCode 通过会话 id 找到上一组提问还没有得到回答的问题
5. 如果问题成功保存到服务，拦截问题后用抛出异常的方式强制掐掉工具执行，并在异常信息中注明：“问题已留言，请稍后，用户将尽快回答”
6. 如果问题保存失败，则打印日志后继续执行工具

### Part 3 技术规范 & 注意事项

1. 除非需求明显模糊或不合理，否则不考虑需求之外未提出要求的实现
2. 你可能发现当前已有一些代码，可以在此基础上进行开发，但**业务逻辑以本文档为准**，如果有多余的代码则直接清理
3. 服务：node & typescript，使用固定端口在 10000 ~ 49152 中随便找一个主流程序不用的即可
4. 插件：node & typescript，OpenCode 插件可以参考 docs 中的文档和 opencode.ai/docs 网站，也可以联网搜索资料
5. 服务要挂载一个数据库，sqlite 关系型数据库，开发自定实体、字段、关系、表结构设计等，满足业务需求即可
6. 以下文件都保存在 **OpenCode 全局配置工程 {env:OPENCODE_CONFIG_DIR} 中**
    1. 数据库要放在 .auto-answer/database 下
    2. **必须有建表语句文件**，放在 .auto-answer/sql 下
7. 可以通过 npm 安装依赖，除了安装依赖之外，**不要修改项目下的配置文件**
8. 所有的 id 字段都使用 uuid，必须有格式校验和必要的存在性校验，如果校验不通过不予处理
9. 可以视 OpenCode 的会话 id 字符串为固定的，直接编码转换为 uuid
10. 始终使用 createLogger() 和 logger.{level}(message) 打印日志而非 console.log，不打印非必要的日志
11. 少而精地使用日志和代码注释
12. 关注 clean code，例如对象、函数职责单一，不要污染全局变量等业界常见规范要求
13. 插件更新不体现在当前会话中，因此直接测试改动是无效的，但你可以通过 bash 下发 opencode run 来启动一个新会话来识别插件改动效果
