# OPENCODE 运行规则

## 重要：当需要你记忆规则时，在此文件下记录

1. 始终回复中文
2. 在编写 AGENTS.md 时（/init、/init-deep 命令触发，或用户手动要求），记录项目关键信息如代码结构、测试框架、测试策略等。
   不要记录 "对新的 opencode session 没有帮助" 的信息，例如：
    - AI 误解或误写代码导致的问题
    - 环境问题 / 依赖问题 / 系统问题等（应由用户处理，AI 不自行处理）
    - 上述问题的处理方案或处理过程
3. 不要记录过期信息。例如原本记录了 A，但后来修改为 B，则直接删除 A，避免新旧信息矛盾造成混淆。
4. 当用户明确指定了某处代码逻辑需要修改时，如果没有明确指定做 "不适用新逻辑的 fallback 操作" 就不要做。fallback 通常不符合用户预期，因为用户既然指定了修改，就期望新逻辑完全替代旧逻辑。
